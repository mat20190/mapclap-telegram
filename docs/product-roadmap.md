# Product roadmap

## Идея MVP

Первый релиз фокусируется на Москве целиком, но с редакционными подборками по районам. Пользователь открывает карту, выбирает категорию и быстро понимает, что можно сделать рядом: спорт, музыка, театр, музеи, еда, развлечения. Маскот Клэп объясняет подборку человеческим языком.

## Почему Expo

Expo дает одну React Native кодовую базу для iOS и Android. Для ранней версии это быстрее и дешевле, чем две отдельные нативные команды. Когда приложение дойдет до публикации, сборки можно делать через EAS Build.

## Наполнение

Сейчас события лежат в `src/data/tagankaGuide.ts`. Это удобно для первого прототипа: можно быстро менять тексты, координаты и категории без сервера.

Следующий этап - вынести контент во внешний источник:

1. Google Sheet или Airtable для ручной редакции.
2. Небольшой backend API для выдачи событий.
3. Модерация и статусы: черновик, опубликовано, архив.
4. Источники с актуальными афишами и ручная редакторская проверка.

## Персонализация

Персонализацию стоит добавлять после базовой карты:

1. Избранное.
2. Фильтры по цене, времени, компании и погоде.
3. Онбординг с интересами.
4. Рекомендации по району, настроению и истории действий.
5. Диалог с Клэпом: "хочу спокойно", "иду на свидание", "есть 2 часа".

## Публикация

Перед App Store и Play Market понадобятся:

1. Реальные `bundleIdentifier` и `android.package`.
2. Иконка, splash screen и скриншоты.
3. Privacy Policy, особенно если будет геолокация и аналитика.
4. Google Maps API key для production-сборок.
5. EAS project и первые test builds.
6. Внутреннее тестирование, исправление крашей и проверка разрешений.
