# Legacy HTML migration

Старый HTML-прототип важен как продуктовый черновик. В нем уже есть базовая логика MapClap:

- Yandex-карта на весь экран.
- Верхние фильтры.
- Карточка выбранного места.
- Режим пользователя.
- Отдельный тип рекламной точки.
- Визуальный маскот как быстрый узнаваемый элемент.

## Как это переносится в мобильное приложение

HTML-прототип был хорош для проверки идеи в браузере. Для App Store и Play Market мы переводим его в Expo/React Native:

- `places` становится типизированным массивом `GuideItem[]`.
- `filter(type)` становится React-состоянием `selectedCategory`.
- `render(data)` заменяется декларативной отрисовкой `Marker` и `GuideCard`.
- `card.style.display = "block"` заменяется выбором карточки и состоянием интерфейса.
- Категории расширяются под продукт: спорт, музыка, театр, музеи, еда, развлечения.

## Что оставить из старого прототипа

Название изменено на `MapClap`: карта плюс быстрый подбор планов по Москве.

Темную, более ночную визуальную тему можно вернуть отдельным режимом позже. Сейчас MVP сделан светлее, потому что на первом этапе важнее читаемость карточек, карты и фильтров.

Рекламные точки лучше не смешивать с пользовательскими категориями. В будущем для них стоит добавить отдельное поле:

```ts
isSponsored: boolean;
```

Так приложение сможет показывать бизнесы нативно, но не ломать доверие к рекомендациям.
